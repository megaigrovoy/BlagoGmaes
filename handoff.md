# 🥷 Ninja Camera Game - Handover Document

## 📝 Описание проекта
**Ninja Camera Game** — это веб-приложение, сочетающее в себе механику легендарной игры Fruit Ninja с современными технологиями компьютерного зрения. Игрок использует свои руки в качестве мечей, чтобы разрезать фрукты-эмодзи, появляющиеся на экране, прямо перед своей веб-камерой.

Проект разработан с упором на эстетику «киберпанк» и плавность взаимодействия в реальном времени.

## 🚀 Технологический стек
*   **Frontend Core**: HTML5, Vanilla JavaScript (ES Modules).
*   **Сборка**: [Vite](https://vitejs.dev/) — для быстрой разработки и оптимизированного билда.
*   **Computer Vision**: [MediaPipe Tasks Vision](https://developers.google.com/mediapipe/solutions/vision) — используется для трекинга рук (`HandLandmarker`) и поз (`PoseLandmarker`).
*   **Графика**: Canvas API — для отрисовки игровых объектов, эффектов частиц и неоновой маски.
*   **Стилизация**: Vanilla CSS — стеклянный эффект (glassmorphism), неоновое свечение и адаптивная верстка.

## ✨ Основные возможности
1.  **Hand Tracking slicing**: Использование суставов пальцев и сегментов ладони в качестве коллийдеров для «разрезания» фруктов.
2.  **Procedural Ninja Mask**: На лицо игрока в реальном времени накладывается процедурно отрисованная маска «Ниндзя-Кота», которая подстраивается под наклон и положение головы с помощью данных `PoseLandmarker`.
3.  **Fruit Physics**: 30% уменьшенные фрукты-эмодзи с реалистичным разделением на две части при разрезе и инерционным вращением.
4.  **Neon Aesthetics**: Темная тонировка камеры, неоновые шлейфы за руками и светящиеся частицы при попадании.
5.  **Cross-Device Testing**: Настроен SSL для корректной работы камеры при тестировании через локальную сеть на мобильных устройствах.

## 📂 Структура проекта
*   `index.html`: Точка входа, структура игрового контейнера и UI-слоя.
*   `main.js`: Основная логика игры: инициализация MediaPipe, игровой цикл (render loop), физика фруктов, обработка коллизий и отрисовка маски.
*   `style.css`: Описание визуального стиля, анимаций загрузки и адаптивности.
*   `vite.config.js`: Конфигурация Vite с поддержкой Basic SSL для доступа к камере по HTTPS.

## 🛠 Запуск проекта
1.  Установите зависимости:
    ```bash
    npm install
    ```
2.  Запустите сервер разработки:
    ```bash
    npm run dev
    ```
3.  Для доступа с мобильного телефона используйте предложенный Vite Network IP адрес (убедитесь, что устройства в одной Wi-Fi сети).

## ⚠️ Текущий статус и советы
*   **Оптимизация**: Модели MediaPipe загружаются через CDN для минимизации размера репозитория.
*   **Масштабирование**: Канвас автоматически подстраивается под размер окна. При изменении ориентации экрана на мобильном устройстве рекомендуется перезагрузить страницу для корректной пересчета пропорций.
*   **Камера**: Требуется разрешение на использование камеры. На некоторых устройствах может потребоваться подтверждение "unsecure certificates" из-за самоподписанного SSL в режиме разработки.
